## S-01. 로그인 화면

### 화면 설명

비인증 사용자가 이메일과 비밀번호를 입력하여 앱에 접근하는 진입 화면이다. 인증에 성공하면 할일 목록 화면(S-03)으로 이동한다. 인증되지 않은 상태에서 보호된 경로에 접근하면 이 화면으로 리다이렉트된다.

### ASCII 와이어프레임

```
+----------------------------------------------------------+
|                                                          |
|                    TodoList                              |
|                                                          |
|  ====================================================== |
|                                                          |
|                       로그인                             |
|                                                          |
|  이메일                                                  |
|  [____________________________________________________]  |
|                                                          |
|  비밀번호                                                |
|  [____________________________________________________]  |
|                                                          |
|  ┌ 오류 메시지 영역 (기본: 숨김) ─────────────────────┐ |
|  │ ! 이메일 또는 비밀번호가 올바르지 않습니다.         │ |
|  └─────────────────────────────────────────────────────┘ |
|                                                          |
|              [        로그인        ]                    |
|                                                          |
|           계정이 없으신가요?  회원가입 →                 |
|                                                          |
+----------------------------------------------------------+
```

### UI 요소 목록

| 요소 | 타입 | 설명 |
|------|------|------|
| 앱 제목 | 텍스트 | "TodoList" 브랜드명 |
| 이메일 입력 | Input (type=email) | 로그인에 사용할 이메일 주소 입력 |
| 비밀번호 입력 | Input (type=password) | 비밀번호 입력 (마스킹 처리) |
| 오류 메시지 영역 | 인라인 알림 | 로그인 실패 시 표시. 기본 숨김 상태 |
| 로그인 버튼 | Button (primary) | 폼 제출. 요청 중 로딩 상태 표시 |
| 회원가입 링크 | 텍스트 링크 | S-02 회원가입 화면으로 이동 |

### 사용자 인터랙션

| 인터랙션 | 동작 |
|----------|------|
| 로그인 버튼 클릭 | 이메일·비밀번호 유효성 검사 → API 호출 → 성공 시 S-03으로 이동, 실패 시 오류 메시지 표시 |
| 회원가입 링크 클릭 | S-02 회원가입 화면으로 이동 |
| Enter 키 입력 | 로그인 버튼 클릭과 동일 |

### 비즈니스 규칙 반영

- 인증되지 않은 사용자가 `/todos`, `/categories` 등 보호 경로 접근 시 이 화면으로 자동 리다이렉트된다 (규칙 3.1).
- 로그인 성공 시 JWT 토큰을 authStore에 저장하고 S-03으로 이동한다.
- 잘못된 이메일/비밀번호 입력 시 오류 메시지를 표시한다 (UC-02 수용 기준).

## S-02. 회원가입 화면

### 화면 설명

신규 사용자가 이름, 이메일, 비밀번호를 입력하여 계정을 생성하는 화면이다. 가입 완료 후 로그인 화면(S-01)으로 이동한다.

### ASCII 와이어프레임

```
+----------------------------------------------------------+
|                                                          |
|                    TodoList                              |
|                                                          |
|  ====================================================== |
|                                                          |
|                      회원가입                            |
|                                                          |
|  이름                                                    |
|  [____________________________________________________]  |
|                                                          |
|  이메일                                                  |
|  [____________________________________________________]  |
|  ┌ 필드 오류 (기본: 숨김) ──────────────────────────┐   |
|  │ ! 이미 사용 중인 이메일입니다.                    │   |
|  └───────────────────────────────────────────────────┘   |
|                                                          |
|  비밀번호                                                |
|  [____________________________________________________]  |
|                                                          |
|              [        가입하기        ]                  |
|                                                          |
|           이미 계정이 있으신가요?  로그인 →              |
|                                                          |
+----------------------------------------------------------+
```

### UI 요소 목록

| 요소 | 타입 | 설명 |
|------|------|------|
| 이름 입력 | Input (type=text) | 사용자 이름 입력 (필수) |
| 이메일 입력 | Input (type=email) | 고유 이메일 주소 입력 (필수) |
| 이메일 오류 메시지 | 인라인 오류 | 중복 이메일 시 필드 하단에 표시 |
| 비밀번호 입력 | Input (type=password) | 비밀번호 입력 (필수, 마스킹 처리) |
| 가입하기 버튼 | Button (primary) | 폼 제출. 요청 중 로딩 상태 표시 |
| 로그인 링크 | 텍스트 링크 | S-01 로그인 화면으로 이동 |

### 사용자 인터랙션

| 인터랙션 | 동작 |
|----------|------|
| 가입하기 버튼 클릭 | 필드 유효성 검사 → API 호출 → 성공 시 S-01로 이동, 실패 시 오류 메시지 표시 |
| 로그인 링크 클릭 | S-01 로그인 화면으로 이동 |

### 비즈니스 규칙 반영

- 이메일은 고유값으로, 이미 등록된 이메일 입력 시 이메일 필드 하단에 중복 오류 메시지를 표시한다 (UC-01 수용 기준).
- 모든 필드(이름, 이메일, 비밀번호)는 필수 입력값이다.
- 가입 성공 후 자동 로그인을 수행하지 않고 S-01로 이동하여 직접 로그인하도록 유도한다.

## S-03. 할일 목록 화면

### 화면 설명

로그인한 사용자 본인의 할일 목록을 조회하는 메인 화면이다. 카테고리, 상태, 기한 초과 기준으로 필터링하여 원하는 항목만 볼 수 있다. 할일 등록, 수정, 삭제, 로그아웃의 주요 진입점 역할도 수행한다.

### ASCII 와이어프레임

```
+----------------------------------------------------------+
|  TodoList                           [내 정보] [환경설정] |
|                               이지은님 안녕하세요 [로그아웃]|
|  ====================================================== |
|                                                          |
|  -- 필터 영역 ----------------------------------------- |
|                                                          |
|  카테고리:  [전체          ▼]                            |
|                                                          |
|  상태:  [■전체■] [대기] [진행중] [완료]                  |
|                                                          |
|  [ ] 기한 초과 미완료만 보기                             |
|                                                          |
|  --------------------------------------------------------|
|                                                          |
|  -- 할일 목록 ----------------------------------------- |
|                                                          |
|  │ [x] 논문 2장 초안 작성          [진행중] [✎] [✕] │  |
|  │     카테고리: 논문 · 2026-06-01 ~ 2026-06-10       │  |
|  ├──────────────────────────────────────────────────────┤  |
|  │ [ ] 아르바이트 스케줄 확인      [대기]   [✎] [✕] │  |
|  │     카테고리: 기본 · 종료일 없음                    │  |
|  ├──────────────────────────────────────────────────────┤  |
|  │ [x] 참고문헌 정리               [완료]   [✎] [✕] │  |
|  │     카테고리: 논문 · 2026-05-20 ~ 2026-05-25       │  |
|  ├──────────────────────────────────────────────────────┤  |
|  │ [!] 팀원 면담 일정              [대기]   [✎] [✕] │  |
|  │     카테고리: 업무 · 기한 초과 (2026-05-25)        │  |
|  └──────────────────────────────────────────────────────┘  |
|                                                          |
|  총 4개 항목                                             |
|                                                          |
|              [+  새 할일 등록  ]                         |
|                                                          |
|  [카테고리 관리]                                         |
|                                                          |
+----------------------------------------------------------+
```

### UI 요소 목록

| 요소 | 타입 | 설명 |
|------|------|------|
| 앱 헤더 | 고정 헤더 | 앱 제목, 사용자 이름, 내 정보/환경설정/로그아웃 버튼 |
| 카테고리 필터 | 드롭다운 | 전체 + 사용자 카테고리 목록. 선택 시 해당 카테고리만 표시 |
| 상태 필터 | 탭 버튼 그룹 | 전체 / 대기 / 진행중 / 완료. 선택된 탭 강조 표시 |
| 기한 초과 필터 | 체크박스 | 체크 시 종료일자 경과 + 미완료 항목만 표시 |
| 할일 목록 | 리스트 | 각 항목: 체크박스, 제목, 상태 배지, 수정/삭제 버튼, 카테고리·날짜 부제 |
| 체크박스 (항목) | 체크박스 | 클릭 시 상태 순환 (TODO → IN_PROGRESS → DONE). DONE 항목은 비활성화 |
| 기한 초과 표시 | 경고 아이콘 [!] | 종료일자가 오늘 이전이고 미완료인 항목에 표시 |
| 수정 버튼 [✎] | 아이콘 버튼 | S-05 할일 수정 화면으로 이동 |
| 삭제 버튼 [✕] | 아이콘 버튼 | 삭제 확인 후 항목 제거 |
| 새 할일 등록 버튼 | Button (primary) | S-04 할일 등록 화면으로 이동 |
| 카테고리 관리 링크 | 텍스트 링크/버튼 | S-06 카테고리 관리 화면으로 이동 |
| 내 정보 버튼 | 텍스트 링크/버튼 | S-07 내 정보 수정 화면으로 이동 |
| 환경설정 버튼 | 텍스트 링크/버튼 | S-08 환경설정 화면으로 이동 (v2) |
| 로그아웃 버튼 | Button (secondary) | authStore 초기화 후 S-01로 이동 |

### 사용자 인터랙션

| 인터랙션 | 동작 |
|----------|------|
| 카테고리 드롭다운 변경 | 선택한 카테고리로 목록 필터링 갱신 |
| 상태 탭 클릭 | 선택한 상태의 할일만 표시, 선택된 탭 강조 |
| 기한 초과 체크박스 토글 | 기한 초과 미완료 항목만 표시/전체 표시 전환 |
| 할일 항목 클릭 (제목 클릭) | S-05 할일 수정 화면으로 이동 |
| [✎] 클릭 | S-05 할일 수정 화면으로 이동 |
| [✕] 클릭 | 삭제 확인 모달 표시 → 확인 시 해당 항목 삭제 |
| [+ 새 할일 등록] 클릭 | S-04 할일 등록 화면으로 이동 |
| [카테고리 관리] 클릭 | S-06 카테고리 관리 화면으로 이동 |
| [내 정보] 클릭 | S-07 내 정보 수정 화면으로 이동 |
| [환경설정] 클릭 | S-08 환경설정 화면으로 이동 |
| [로그아웃] 클릭 | 인증 세션 종료 후 S-01로 이동 |

### 비즈니스 규칙 반영

- 로그인한 사용자 본인의 할일만 표시한다 (규칙 3.2).
- 기한 초과 미완료 필터: 종료일자 < 오늘 날짜이고 상태가 DONE이 아닌 항목 (규칙 3.5).
- DONE 상태의 할일은 목록 체크박스를 비활성화하여 상태 역전이를 UI에서 차단한다 (규칙 4.2).
- 삭제 시 확인 절차를 거쳐 실수 삭제를 방지한다 (UC-07).

## S-04. 할일 등록 화면

### 화면 설명

새로운 할일을 등록하는 화면이다. 제목은 필수이며, 카테고리·시작일자·종료일자·상세 내용은 선택 입력이다. 등록 완료 후 할일 목록 화면(S-03)으로 이동한다.

### ASCII 와이어프레임

```
+----------------------------------------------------------+
|  ← 뒤로                    새 할일 등록                  |
|  ====================================================== |
|                                                          |
|  제목 *                                                  |
|  [____________________________________________________]  |
|                                                          |
|  카테고리                                                |
|  [기본            ▼]                                    |
|  (미선택 시 '기본' 카테고리 자동 적용)                   |
|                                                          |
|  시작일자                          종료일자              |
|  [  YYYY-MM-DD  📅 ]              [  YYYY-MM-DD  📅 ]  |
|                                                          |
|  ┌ 날짜 오류 (기본: 숨김) ──────────────────────────┐   |
|  │ ! 종료일자는 시작일자 이후여야 합니다.             │   |
|  └───────────────────────────────────────────────────┘   |
|                                                          |
|  상세 내용 (선택)                                        |
|  +────────────────────────────────────────────────────+ |
|  |                                                    | |
|  |                                                    | |
|  |                                                    | |
|  +────────────────────────────────────────────────────+ |
|                                                          |
|  * 필수 입력 항목                                        |
|                                                          |
|  [        취소        ]  [        등록        ]          |
|                                                          |
+----------------------------------------------------------+
```

### UI 요소 목록

| 요소 | 타입 | 설명 |
|------|------|------|
| 뒤로 버튼 | 텍스트 링크 | S-03 할일 목록 화면으로 이동 (저장하지 않음) |
| 제목 입력 | Input (type=text) | 할일 제목 입력. 필수 항목 |
| 카테고리 드롭다운 | Select | 기본 카테고리 포함 전체 카테고리 목록. 미선택 시 '기본' 자동 적용 |
| 시작일자 선택 | DatePicker | 캘린더 UI로 날짜 선택. 선택 취소 가능 |
| 종료일자 선택 | DatePicker | 캘린더 UI로 날짜 선택. 시작일자보다 이전 날짜 선택 불가 |
| 날짜 오류 메시지 | 인라인 오류 | 종료일자 < 시작일자일 때 표시 |
| 상세 내용 입력 | Textarea | 할일 부연 설명 입력. 선택 항목 |
| 취소 버튼 | Button (secondary) | S-03으로 이동 (변경 사항 저장 안 함) |
| 등록 버튼 | Button (primary) | 유효성 검사 후 API 호출하여 할일 등록 |

### 사용자 인터랙션

| 인터랙션 | 동작 |
|----------|------|
| 시작일자 📅 클릭 | 캘린더 UI 표시하여 날짜 선택 |
| 종료일자 📅 클릭 | 캘린더 UI 표시. 시작일자 선택 시 시작일자 이전 날짜는 선택 불가 처리 |

- 사용자는 자신이 생성한 카테고리만 수정, 삭제할 수 있다 (규칙 3.3).
- 카테고리 목록은 시스템 기본 카테고리를 최상단에 고정하여 표시한다.

| 저장 완료 메시지 | 성공 알림 | 저장 성공 시 표시. 기본 숨김 상태 |
| 저장 버튼 | Button (primary) | 변경된 테마/언어를 API로 저장 |

### 사용자 인터랙션

| 인터랙션 | 동작 |
|----------|------|
| Light Mode / Dark Mode 라디오 클릭 | uiStore의 theme 상태 변경 → 앱 전체 테마 즉시 전환 |
| 언어 드롭다운 변경 | 선택한 언어로 i18n 전환 → 앱 전체 텍스트 즉시 전환 |
| 저장 버튼 클릭 | API 호출 (PATCH /api/users/me/preferences) → 성공 시 완료 메시지 표시 |
| 뒤로 버튼 클릭 | S-03 할일 목록 화면으로 이동 |

### 비즈니스 규칙 반영

- 테마 변경 시 UI가 즉시 적용되며, 저장 버튼 클릭 시 사용자 정보에 영구 저장된다 (규칙 3.6, UC-13).
- 저장된 테마는 다음 로그인 시 자동으로 적용된다 (규칙 3.6).
- 언어 변경 시 앱 전체 텍스트가 즉시 전환된다 (규칙 3.7, UC-12).
- 비인증 상태(로그인·회원가입 화면)에서도 언어 선택이 적용된다 (규칙 3.7).
- LIGHT/DARK 외의 값은 저장되지 않도록 유효성 검사를 수행한다 (규칙 3.6).
- 미설정 시 기본값은 Light Mode, 언어는 한국어(ko)이다 (규칙 3.6, 3.7).

### 공통 버튼 유형

| 유형 | 사용처 | 스타일 |
|------|--------|--------|
| Primary | 주요 액션 (등록, 저장, 로그인) | 강조 색상 |
| Secondary | 보조 액션 (취소, 뒤로) | 아웃라인 또는 회색 |
| Danger | 삭제 확인 | 빨간색 계열 |
| Disabled | 비활성화 (기본 카테고리 수정/삭제, DONE 상태 전이) | 회색, 클릭 불가 |

### 공통 오류 처리 패턴

| 오류 유형 | 표시 위치 | 표시 방식 |
|----------|-----------|-----------|
| 필드 유효성 오류 | 해당 필드 하단 | 인라인 오류 메시지 (빨간색 텍스트) |
| API 오류 (인증 실패 등) | 폼 상단 | 오류 박스 (노란색 또는 빨간색 박스) |
| 성공 완료 | 폼 내부 또는 상단 | 성공 메시지 박스 (초록색 텍스트) |
| 삭제 확인 | 모달 다이얼로그 | 확인 / 취소 버튼 포함 |

### 반응형 고려사항
