## 1. 제품 개요

### 1.1 목적

인증된 사용자가 개인별로 할일을 등록, 조회, 수정, 삭제할 수 있는 웹 기반 할일 관리 애플리케이션이다. 사용자 인증을 통해 개인 데이터를 분리하고, 시작일과 종료일 기반의 일정 관리 기능을 제공한다.

### 1.2 목표 사용자

- 학생
- 20대~50대 직장인

### 1.3 핵심 가치

- 사용자별 데이터 격리: 로그인한 사용자만 자신의 할일에 접근 가능
- 일정 기반 관리: 시작일, 종료일로 할일 기간 추적
- 카테고리 분류: 기본 카테고리 및 사용자 정의 카테고리로 할일 분류
- 상태 관리: 대기 > 진행중 > 완료 단계별 진행 상태 추적

## 3. 기능 요구사항 (MVP)

도메인 정의서 v2.0의 유스케이스 전체가 MVP 범위에 해당한다.

### UC-01. 회원가입

- 이메일, 비밀번호, 이름을 입력하여 계정을 생성한다.
- 이메일은 중복 불가(고유값)이다.

### UC-02. 로그인

- 이메일과 비밀번호로 인증하여 앱에 접근한다.
- 인증되지 않은 사용자가 앱 기능에 접근하면 로그인 화면으로 리다이렉트된다.

### UC-03. 내 정보 수정

- 인증된 사용자는 자신의 이름과 비밀번호를 수정할 수 있다.
- 수정 가능한 필드: 이름, 비밀번호

### UC-04. 할일 등록

- 제목(필수), 카테고리(선택), 시작일자(선택), 종료일자(선택), 상세 내용(선택)을 입력하여 할일을 등록한다.
- 카테고리 미지정 시 '기본(DEFAULT)' 카테고리가 자동 적용된다.
- 시작일자, 종료일자는 캘린더 UI로 선택한다.
- 종료일자는 시작일자보다 같거나 이후 날짜여야 한다.
- 초기 상태는 '대기(TODO)'이다.

### UC-05. 할일 목록 조회

- 로그인한 사용자 본인의 할일만 조회된다.
- 아래 기준으로 필터링할 수 있다.
  - 카테고리별
  - 상태별: 대기 / 진행중 / 완료
  - 기한 초과 미완료 (종료일자가 오늘 이전이고 상태가 완료가 아닌 항목)

### UC-06. 할일 수정

- 편집 화면에서 제목, 상세 내용, 시작일자, 종료일자, 상태, 카테고리를 수정할 수 있다.
- 상태 전이 규칙을 준수한다.
  - 대기 → 진행중 → 완료
  - 진행중 → 대기 되돌리기 허용
  - 완료 상태에서 다른 상태로의 전이 불가

### UC-07. 할일 삭제

- 로그인한 사용자는 자신의 할일을 삭제할 수 있다.
- 타인의 할일 삭제 시도 시 접근 거부 오류를 반환한다.

### UC-08. 카테고리 등록

- 인증된 사용자는 카테고리 이름을 입력하여 새 카테고리를 생성할 수 있다.

### UC-09. 카테고리 수정

- 인증된 사용자는 자신이 생성한 카테고리의 이름을 수정할 수 있다.
- '기본(DEFAULT)' 카테고리는 수정할 수 없다.

### UC-10. 카테고리 삭제

- 인증된 사용자는 자신이 생성한 카테고리를 삭제할 수 있다.
- '기본(DEFAULT)' 카테고리는 삭제할 수 없다.

### UC-11. 로그아웃

- 인증된 사용자는 현재 인증 세션을 종료하고 로그아웃할 수 있다.
- 로그아웃 후 로그인 화면으로 이동된다.

### UC-12. 다국어 지원 _(v2)_

- 사용자는 앱 사용 언어를 선택할 수 있다.
- 선택한 언어는 사용자 정보에 저장되어 다음 로그인 시에도 유지된다.
- 비인증 상태(로그인·회원가입 화면)에서도 언어 선택이 적용된다.

### UC-13. 테마 모드 설정 _(v2)_

- 사용자는 Dark Mode / Light Mode를 선택할 수 있다.
- 선택한 테마는 사용자 정보에 저장되어 다음 로그인 시 자동으로 적용된다.
- 미설정 시 기본값은 Light Mode이다.

## 4. 비기능 요구사항

### 4.1 성능

- 동시 접속자 1,000명 지원
- API 응답 시간 p95 기준 500ms 이하

### 4.2 보안

- 비밀번호는 암호화하여 저장한다 (평문 저장 금지).
- 인증 토큰 기반 세션 관리를 사용한다.
- 사용자는 자신의 리소스(할일, 카테고리)에만 접근 가능하다.

### 4.3 기술 스택 제약

| 영역 | 기술 |
|------|------|
| 프론트엔드 | React 19, TypeScript, Zustand, TanStack Query |
| 다국어 (v2) | react-i18next |
| 백엔드 | Node.js, JavaScript, Express, pg 라이브러리 |
| 인증 | JWT (Bearer Token) |
| 데이터베이스 | PostgreSQL 17 |
| ORM | 사용 금지 (Prisma 금지, pg 직접 사용) |
| 플랫폼 | 웹 우선, 모바일은 반응형 웹 UI |

## 5. 시스템 아키텍처 개요

### 5.1 전체 구성

3-Tier 구조로 프론트엔드, 백엔드, 데이터베이스가 분리된다.

### 5.2 프론트엔드

- React 19 + TypeScript 기반 SPA(Single Page Application)
- Zustand로 클라이언트 상태(인증 정보, UI 상태) 관리
- TanStack Query로 서버 데이터 fetching 및 캐싱 처리
- 반응형 UI로 모바일 브라우저 대응

### 5.3 백엔드

- Node.js + Express 기반 REST API 서버
- pg 라이브러리로 PostgreSQL 직접 쿼리
- JWT(Bearer Token) 기반 인증 처리
- 요청마다 사용자 인증 및 리소스 소유권 검증

### 5.4 데이터베이스

- PostgreSQL 17
- 테이블: users, categories, todos
- categories 테이블에 user_id = NULL인 시스템 전역 기본 카테고리 사전 등록
- users 테이블에 `theme` (LIGHT/DARK, 기본값 LIGHT), `language` (기본값 'ko') 컬럼 추가 _(v2)_

## 7. API 설계 방향

RESTful 원칙에 따라 설계하며, 모든 인증 필요 API는 Authorization 헤더를 사용한다.

### 7.1 인증

| 메서드 | 경로 | 설명 |
|--------|------|------|
| POST | /api/auth/register | 회원가입 |
| POST | /api/auth/login | 로그인 |
| POST | /api/auth/logout | 로그아웃 |

### 7.2 사용자

| 메서드 | 경로 | 설명 |
|--------|------|------|
| PATCH | /api/users/me | 내 정보 수정 (이름, 비밀번호) |
| PATCH | /api/users/me/preferences | 환경설정 저장 (테마, 언어) _(v2)_ |
| GET | /api/users/me | 내 정보 조회 (테마, 언어 포함) _(v2)_ |

### 7.3 카테고리

| 메서드 | 경로 | 설명 |
|--------|------|------|
| GET | /api/categories | 카테고리 목록 조회 (시스템 + 본인 카테고리) |
| POST | /api/categories | 카테고리 등록 |
| PATCH | /api/categories/:id | 카테고리 수정 |
| DELETE | /api/categories/:id | 카테고리 삭제 |

### 7.4 할일

| 메서드 | 경로 | 설명 |
|--------|------|------|
| GET | /api/todos | 할일 목록 조회 (필터 쿼리 파라미터 지원) |
| POST | /api/todos | 할일 등록 |
| PATCH | /api/todos/:id | 할일 수정 |
| DELETE | /api/todos/:id | 할일 삭제 |

## 8. 개발 일정 (2일)

### Day 1 - 백엔드 및 인증 기반 구축

- PostgreSQL DB 스키마 생성 (users, categories, todos 테이블)
- 기본(DEFAULT) 카테고리 초기 데이터 삽입

- 프로젝트 초기 설정 (React 19, TypeScript, Zustand, TanStack Query)
- 로그인 / 회원가입 화면 구현
- 인증 상태 관리 및 라우팅 가드 구현
- 할일 목록 화면 구현 (필터 포함)
- 할일 등록 / 수정 화면 구현 (캘린더 포함)
- 카테고리 관리 화면 구현
