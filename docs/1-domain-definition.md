## 1. 개요

### 앱 목적
인증된 사용자가 개인별로 할일을 등록, 조회, 수정, 삭제할 수 있는 할일 관리 애플리케이션이다.

### 핵심 문제
- 사용자 인증 없이 운영되는 할일 관리 앱은 개인 데이터 분리가 불가능하다.
- 기존 할일 관리 앱은 시작일과 종료일 기반의 일정 관리 기능이 부족하다.

## 2. 도메인 모델

### 2.1 사용자 (User)

| 속성 | 타입 | 설명 |
|------|------|------|
| id | 고유 식별자 | 사용자 고유 ID |
| email | 문자열 | 로그인에 사용하는 이메일 (고유값) |
| password | 문자열 | 암호화된 비밀번호 |
| name | 문자열 | 사용자 이름 |
| created_at | 일시 | 가입일시 |
| updated_at | 일시 | 정보 수정일시 |
| theme | 열거형 | UI 테마 모드 (LIGHT/DARK, 기본값 LIGHT) _(v2)_ |
| language | 문자열 | 앱 사용 언어 코드 (기본값 'ko') _(v2)_ |

### 2.2 카테고리 (Category)

| 속성 | 타입 | 설명 |
|------|------|------|
| id | 고유 식별자 | 카테고리 고유 ID |
| user_id | 고유 식별자 (NULL 허용) | 소유 사용자 ID. NULL이면 시스템 전역 기본 카테고리 |
| name | 문자열 | 카테고리 이름 |
| created_at | 일시 | 등록일시 |

> '기본(DEFAULT)' 카테고리는 user_id = NULL인 시스템 전역 카테고리로, 모든 사용자에게 공통으로 적용된다. 사용자가 생성한 카테고리는 user_id에 해당 사용자의 ID가 저장된다.

### 2.3 할일 (Todo)

| 속성 | 타입 | 설명 |
|------|------|------|
| id | 고유 식별자 | 할일 고유 ID |
| user_id | 고유 식별자 | 소유 사용자 ID |
| category_id | 고유 식별자 | 카테고리 ID (미지정 시 '기본' 카테고리) |
| title | 문자열 | 할일 제목 |
| description | 문자열 | 할일 상세 내용 (선택) |
| start_date | 날짜 | 시작일자 (선택) |
| end_date | 날짜 | 종료일자 (선택) |
| status | 열거형 | 할일 상태 |
| created_at | 일시 | 등록일시 |
| updated_at | 일시 | 수정일시 |

## 3. 비즈니스 규칙

### 3.1 인증 규칙
- 회원가입 후 로그인한 인증된 사용자만 앱을 사용할 수 있다.
- 사용자는 자신의 이름과 비밀번호를 수정할 수 있다.
- **규칙 위반 시**: 인증되지 않은 사용자가 앱 기능에 접근하면 로그인 화면으로 리다이렉트된다.

### 3.2 할일 소유권 규칙
- 할일은 반드시 특정 사용자에게 귀속된다.
- 사용자는 자신의 할일만 조회, 수정, 삭제할 수 있다.
- **규칙 위반 시**: 타인의 할일에 접근하면 접근 거부 오류를 반환한다.

### 3.3 카테고리 규칙
- 할일 등록 시 카테고리를 지정하지 않으면 '기본(DEFAULT)' 카테고리가 자동 적용된다.
- '기본(DEFAULT)' 카테고리는 수정하거나 삭제할 수 없다.
- 사용자는 자신이 생성한 카테고리만 수정, 삭제할 수 있다.
- **규칙 위반 시**: '기본(DEFAULT)' 카테고리 수정/삭제 시도 시 오류를 반환한다.

### 3.4 일자 규칙
- 시작일자와 종료일자는 캘린더를 통해 선택한다.
- 종료일자는 시작일자보다 같거나 이후 날짜여야 한다.
- **규칙 위반 시**: 종료일자가 시작일자보다 이전인 경우 저장을 거부하고 오류 메시지를 표시한다.

### 3.6 테마 모드 규칙 _(v2)_
- 사용자는 Dark Mode / Light Mode 중 하나를 선택할 수 있다.
- 선택한 테마는 User 엔티티의 `theme` 필드에 저장되어 다음 로그인 시 자동으로 적용된다.
- 미설정 시 기본값은 LIGHT이다.
- **규칙 위반 시**: LIGHT/DARK 외의 값은 저장을 거부한다.

### 3.7 다국어 규칙 _(v2)_
- 사용자는 앱 사용 언어를 선택할 수 있다.
- 선택한 언어는 User 엔티티의 `language` 필드에 저장되어 다음 로그인 시 자동으로 적용된다.
- 비인증 상태(로그인·회원가입 화면)에서도 언어 선택이 적용된다.
- 미설정 시 기본값은 'ko'(한국어)이다.

### 3.5 필터링 규칙
할일 목록 조회 시 다음 기준으로 필터링할 수 있다.

| 필터 | 조건 |
|------|------|
| 카테고리별 | 지정한 카테고리에 속한 할일 |
| 시작하지 않은 할일 | 상태가 '대기'인 할일 |
| 진행중인 할일 | 상태가 '진행중'인 할일 |
| 완료된 할일 | 상태가 '완료'인 할일 |
| 기한 초과 미완료 | 종료일자가 오늘보다 이전이고 상태가 '완료'가 아닌 할일 |

## 4. 할일 상태 정의

### 4.1 상태 목록

| 상태 | 값 | 설명 |
|------|----|------|
| 대기 | TODO | 아직 시작하지 않은 할일 |
| 진행중 | IN_PROGRESS | 현재 진행 중인 할일 |
| 완료 | DONE | 완료된 할일 |

### 4.2 상태 전이 조건

```
[대기] ---> [진행중] ---> [완료]
  ^              |
  |              |
  +<-------------+  (진행중 -> 대기 되돌리기 허용)
```

| 전이 | 조건 |
|------|------|
| 대기 → 진행중 | 사용자가 직접 상태 변경 |
| 진행중 → 완료 | 사용자가 직접 상태 변경 |
| 진행중 → 대기 | 사용자가 직접 상태 되돌리기 |

> 완료 상태에서 다른 상태로의 전이는 허용하지 않는다.

### 유스케이스별 핵심 수용 기준

**UC-01 회원가입**
- Given: 비인증 사용자가 회원가입 화면에 접근했을 때
  When: 유효한 이메일, 비밀번호, 이름을 입력하고 가입을 요청하면
  Then: 계정이 생성되고 로그인 화면으로 이동한다.
- Given: 비인증 사용자가 회원가입을 시도할 때
  When: 이미 등록된 이메일을 입력하면
  Then: 중복 이메일 오류 메시지를 표시하고 계정을 생성하지 않는다.

**UC-02 로그인**
- Given: 비인증 사용자가 로그인 화면에 있을 때
  When: 등록된 이메일과 올바른 비밀번호를 입력하면
  Then: 인증에 성공하고 할일 목록 화면으로 이동한다.
- Given: 비인증 사용자가 로그인을 시도할 때
  When: 잘못된 비밀번호를 입력하면
  Then: 인증 실패 오류 메시지를 표시한다.

**UC-03 내 정보 수정**
- Given: 인증된 사용자가 내 정보 수정 화면에 있을 때
  When: 새로운 이름 또는 비밀번호를 입력하고 저장하면
  Then: 해당 정보가 변경된다.

**UC-04 할일 등록**
- Given: 인증된 사용자가 할일 등록 화면에 있을 때
  When: 카테고리를 지정하지 않고 할일을 등록하면
  Then: '기본(DEFAULT)' 카테고리가 자동으로 적용되어 저장된다.
- Given: 인증된 사용자가 할일 등록 화면에 있을 때
  When: 종료일자를 시작일자보다 이전 날짜로 입력하면
  Then: 저장이 거부되고 일자 오류 메시지를 표시한다.

**UC-05 할일 목록 조회**
- Given: 인증된 사용자가 할일 목록 화면에 있을 때
  When: 특정 카테고리 필터를 선택하면
  Then: 해당 카테고리에 속한 자신의 할일만 표시된다.

**UC-06 할일 수정**
- Given: 인증된 사용자가 자신의 할일 편집 화면에 있을 때
  When: 상태를 '완료'로 변경하면
  Then: 상태가 '완료'로 저장되고 이후 다른 상태로 변경할 수 없다.

**UC-07 할일 삭제**
- Given: 인증된 사용자가 자신의 할일 목록에 있을 때
  When: 특정 할일 삭제를 요청하면
  Then: 해당 할일이 목록에서 제거된다.

**UC-08 카테고리 등록**
- Given: 인증된 사용자가 카테고리 등록 화면에 있을 때
  When: 카테고리 이름을 입력하고 저장하면

**UC-09 카테고리 수정**

  Then: 카테고리 이름이 변경된다.
- Given: 인증된 사용자가 카테고리 수정을 시도할 때
  When: '기본(DEFAULT)' 카테고리를 수정하려 하면
  Then: 수정이 거부되고 오류 메시지를 표시한다.

**UC-10 카테고리 삭제**

  When: 삭제를 확인하면
  Then: 해당 카테고리가 삭제된다.
- Given: 인증된 사용자가 카테고리 삭제를 시도할 때
  When: '기본(DEFAULT)' 카테고리를 삭제하려 하면
  Then: 삭제가 거부되고 오류 메시지를 표시한다.

**UC-11 로그아웃**
- Given: 인증된 사용자가 앱을 사용 중일 때
  When: 로그아웃을 요청하면
  Then: 인증 세션이 종료되고 로그인 화면으로 이동한다.

**UC-12 다국어 지원** _(v2)_
- Given: 사용자가 언어 선택 화면에 있을 때
  When: 특정 언어를 선택하면
  Then: 앱 전체 텍스트가 선택한 언어로 전환된다.
- Given: 인증된 사용자가 언어를 변경하면
  When: 저장을 요청하면
  Then: 선택한 언어가 사용자 정보에 저장되어 다음 로그인 시에도 유지된다.

**UC-13 테마 모드 설정** _(v2)_
- Given: 인증된 사용자가 환경설정 화면에 있을 때
  When: Dark Mode 또는 Light Mode를 선택하면
  Then: 앱 UI가 즉시 선택한 테마로 전환되고 사용자 정보에 저장된다.
- Given: 인증된 사용자가 다음에 로그인했을 때
  When: 이전에 테마를 설정한 경우
  Then: 저장된 테마 모드가 자동으로 적용된다.

## 6. 용어 사전 (Glossary)

| 용어 | 정의 |
|------|------|
| 기본 카테고리 | user_id = NULL인 시스템 전역 카테고리. 할일 등록 시 카테고리를 지정하지 않으면 자동으로 적용되며, 수정 및 삭제가 불가능하다. |
| 인증된 사용자 | 회원가입 후 로그인하여 유효한 JWT 토큰을 보유한 사용자. 인증된 사용자만 앱의 주요 기능을 사용할 수 있다. 여기서 "인증 세션"은 서버 세션이 아닌 JWT(Bearer Token) 기반 인증 상태를 의미한다. |
| 기한 초과 | 할일의 종료일자가 오늘 날짜보다 이전이고, 상태가 '완료(DONE)'가 아닌 상태를 의미한다. |
| 상태 | 할일의 진행 단계를 나타내는 열거형 값으로 대기(TODO), 진행중(IN_PROGRESS), 완료(DONE) 세 가지가 존재한다. |
| 테마 모드 | UI의 색상 테마를 나타내는 열거형 값으로 LIGHT(밝은 화면)와 DARK(어두운 화면) 두 가지가 존재한다. 기본값은 LIGHT이다. _(v2)_ |
| 언어 코드 | 앱 표시 언어를 나타내는 문자열 값으로 ISO 639-1 코드를 사용한다 (예: 'ko', 'en'). 기본값은 'ko'이다. _(v2)_ |
