## 페르소나 A — 이지은 (25세, 대학원생)

### 시나리오 US-01. 최초 사용

**배경**
기말 논문 제출 시즌이 다가오면서 논문 마감일과 아르바이트 스케줄이 뒤엉켜 무엇부터 해야 할지 모르게 된 이지은은 지인 추천으로 TodoList 앱을 처음 접한다.

**목표**
앱에 계정을 만들고 로그인한 뒤, 가장 급한 논문 관련 할일을 첫 번째로 등록한다.

**행동 흐름**
1. 앱에 접속하면 로그인 화면(S-01)이 나타난다.
2. '회원가입' 링크를 눌러 회원가입 화면(S-02)으로 이동한다.
3. 이름(이지은), 이메일, 비밀번호를 입력하고 가입 버튼을 누른다.
4. 가입 완료 후 로그인 화면(S-01)으로 돌아온다.
5. 이메일과 비밀번호를 입력하고 로그인한다.
6. 할일 목록 화면(S-03)으로 이동한다. 목록은 비어 있다.
7. 새 할일 등록 버튼을 눌러 할일 등록 화면(S-04)으로 이동한다.
8. 제목에 "논문 2장 초안 작성"을 입력하고, 시작일자와 종료일자를 캘린더 UI로 선택한다.
9. 상세 내용에 "지도교수님 피드백 반영 후 제출"을 입력한다.
10. 등록 버튼을 누른다.

**결과**
할일 목록 화면(S-03)에 "논문 2장 초안 작성" 항목이 '대기' 상태로 나타난다. 이지은은 앱 사용을 시작할 준비가 되었다.

---

### 시나리오 US-02. 일상 사용

**배경**
앱을 사용한 지 일주일이 지난 이지은은 등록해 둔 여러 할일 중에서 현재 진행 중인 것과 아직 시작 못한 것을 빠르게 구분하고 싶다. 논문 작업은 이미 시작했으므로 상태를 '진행중'으로 바꾸려 한다.

**목표**
할일 목록에서 상태별 필터를 활용해 원하는 항목을 찾고, 상태를 변경한다.

**행동 흐름**
1. 로그인 후 할일 목록 화면(S-03)에서 등록된 할일 전체를 확인한다.
2. 상태 필터에서 '대기'를 선택한다. 아직 시작하지 않은 할일만 표시된다.
3. "논문 2장 초안 작성" 항목을 선택해 할일 수정 화면(S-05)으로 이동한다.
4. 상태를 '대기'에서 '진행중'으로 변경하고 저장한다.
5. 목록 화면으로 돌아오면 해당 항목이 '대기' 필터에서 사라진 것을 확인한다.
6. 상태 필터를 '진행중'으로 변경하면 방금 수정한 항목이 나타난다.

**결과**
이지은은 현재 진행 중인 할일과 아직 시작 못한 할일을 상태 필터로 명확히 구분해서 볼 수 있게 되었다.

---

### 시나리오 US-03. 카테고리 관리

**배경**
할일이 늘어나면서 논문 관련 항목과 아르바이트 관련 항목이 섞여 보이기 시작했다. 이지은은 두 영역을 카테고리로 나눠 관리하기로 한다.

**목표**
'논문'과 '아르바이트' 카테고리를 만들고, 기존 할일에 카테고리를 적용한 뒤 카테고리 필터로 원하는 항목만 조회한다.

**행동 흐름**
1. 카테고리 관리 화면(S-06)으로 이동한다.
2. '논문' 카테고리 이름을 입력하고 추가한다.
3. '아르바이트' 카테고리 이름을 입력하고 추가한다.
4. 할일 목록 화면(S-03)으로 돌아와 "논문 2장 초안 작성" 항목을 선택한다.
5. 할일 수정 화면(S-05)에서 카테고리를 '논문'으로 변경하고 저장한다.
6. 같은 방법으로 아르바이트 관련 할일에 '아르바이트' 카테고리를 적용한다.
7. 할일 목록 화면(S-03)에서 카테고리 필터를 '논문'으로 설정한다.

**결과**
논문 관련 할일만 목록에 표시된다. 이지은은 논문 진행 상황을 한눈에 파악할 수 있게 되었다.

---

### 시나리오 US-04. 예외 상황 — 기한 초과 할일 처리

**배경**
바쁜 일주일을 보낸 이지은이 앱을 열어보니 종료일자가 지났는데도 완료하지 못한 할일이 있다.

**목표**
기한이 지난 미완료 항목을 확인하고, 종료일자를 연장하거나 상태를 적절히 처리한다.

**행동 흐름**
1. 할일 목록 화면(S-03)에서 필터를 '기한 초과 미완료'로 설정한다.
2. "논문 참고문헌 정리" 항목이 표시된다.
3. 해당 항목을 선택해 할일 수정 화면(S-05)으로 이동한다.
4. 종료일자를 오늘로부터 3일 뒤로 변경하고 저장한다.
5. 목록 화면으로 돌아오면 해당 항목이 '기한 초과 미완료' 필터에서 사라진 것을 확인한다.

**결과**
기한 초과 항목이 정리되었고, 이지은은 현실적인 마감일로 일정을 재조정했다.

## 페르소나 B — 김민준 (38세, 직장인 팀장)

### 시나리오 US-05. 최초 사용

**배경**
팀을 맡게 된 후 업무 지시, 보고 준비, 팀 회식 준비 등 개인 일정까지 뒤섞여 머릿속이 복잡해진 김민준은 앱을 통해 정리를 시작하기로 한다.

**목표**
계정을 만들고 로그인한 뒤, 가장 긴급한 업무 할일을 등록한다.

**행동 흐름**
1. 앱에 접속하면 로그인 화면(S-01)이 나타난다.
2. '회원가입' 링크를 눌러 회원가입 화면(S-02)으로 이동한다.
3. 이름(김민준), 이메일, 비밀번호를 입력하고 가입 버튼을 누른다.
4. 가입 완료 후 로그인 화면(S-01)으로 돌아와 로그인한다.
5. 할일 목록 화면(S-03)으로 이동한다.
6. 새 할일 등록 버튼을 눌러 할일 등록 화면(S-04)으로 이동한다.
7. 제목에 "2분기 팀 성과 보고서 작성"을 입력하고, 종료일자를 이번 주 금요일로 선택한다.
8. 상세 내용에 "부서장 보고 전 목요일까지 초안 완성"을 입력한다.
9. 등록 버튼을 누른다.

**결과**
할일 목록 화면(S-03)에 "2분기 팀 성과 보고서 작성" 항목이 '대기' 상태로 나타난다.

### 시나리오 US-06. 일상 사용

**배경**
월요일 아침, 김민준은 이번 주에 처리해야 할 업무 할일을 점검하고, 어제 퇴근 전에 시작한 보고서 항목을 '진행중'으로 업데이트하려 한다.

**목표**
목록 전체를 빠르게 훑어보고 진행 상태를 최신 상태로 갱신한다.

**행동 흐름**
1. 로그인 후 할일 목록 화면(S-03)에서 전체 할일을 확인한다.
2. 상태 필터를 '대기'로 설정해 아직 시작하지 않은 항목만 본다.
3. "2분기 팀 성과 보고서 작성" 항목을 선택해 할일 수정 화면(S-05)으로 이동한다.
4. 상태를 '진행중'으로 변경하고 저장한다.
5. 다시 상태 필터를 '진행중'으로 변경해 현재 진행 중인 할일 목록을 확인한다.

**결과**
김민준은 현재 진행 중인 업무 목록을 한눈에 파악하고 오늘의 우선순위를 정할 수 있게 되었다.

### 시나리오 US-07. 카테고리 관리

**배경**
할일이 10개를 넘어가자 업무 항목과 개인 항목이 뒤섞여 불편함을 느낀 김민준은 카테고리로 구분하기로 한다.

**목표**
'업무'와 '개인' 카테고리를 생성하고, 기존 할일에 적용한 뒤 카테고리 필터로 업무만 조회한다.

**행동 흐름**
1. 카테고리 관리 화면(S-06)으로 이동한다.
2. '업무' 카테고리를 추가한다.
3. '개인' 카테고리를 추가한다.
4. 할일 목록 화면(S-03)으로 돌아와 "2분기 팀 성과 보고서 작성" 항목을 선택한다.
5. 할일 수정 화면(S-05)에서 카테고리를 '업무'로 변경하고 저장한다.
6. 나머지 업무 관련 할일에도 동일하게 '업무' 카테고리를 적용한다.
7. 개인 일정 할일에는 '개인' 카테고리를 적용한다.
8. 할일 목록 화면(S-03)에서 카테고리 필터를 '업무'로 설정한다.

**결과**
업무 관련 할일만 표시된다. 김민준은 업무와 개인 일정을 명확히 분리해서 관리할 수 있게 되었다.

---

### 시나리오 US-08. 예외 상황 — 기한 초과 할일 처리

**배경**
금요일 퇴근 후 앱을 확인한 김민준이 이번 주 마감이었던 할일 중 완료하지 못한 항목이 있음을 발견한다.

**목표**
기한 초과 미완료 항목을 파악하고 다음 주로 일정을 조정하거나 삭제 여부를 결정한다.

**행동 흐름**
1. 할일 목록 화면(S-03)에서 필터를 '기한 초과 미완료'로 설정한다.
2. "팀원 개별 면담 일정 잡기" 항목이 표시된다.
3. 해당 항목을 선택해 할일 수정 화면(S-05)으로 이동한다.
4. 종료일자를 다음 주 수요일로 변경하고 저장한다.
5. 할일 목록 화면(S-03)으로 돌아와 더 이상 '기한 초과 미완료' 필터에 항목이 없는 것을 확인한다.

**결과**
기한 초과 항목이 정리되었고, 김민준은 다음 주 일정을 현실적으로 재배분했다.

#### 페르소나 A — 이지은: 다크 모드 전환 및 언어 변경

**배경**
밤늦게 논문 작업을 하던 이지은은 화면이 너무 밝아 눈이 피로하다. 또한 영어 공부를 겸해 앱 언어를 영어로 바꾸고 싶다.

**목표**
다크 모드로 전환하고 앱 언어를 영어로 변경한 뒤, 다음에 로그인해도 설정이 유지되는지 확인한다.

**행동 흐름**
1. 환경설정 화면(S-08)으로 이동한다.
2. 테마 모드를 'Dark Mode'로 변경한다. 화면이 즉시 어두운 테마로 바뀐다.
3. 언어 설정에서 'English'를 선택한다. 앱 전체 텍스트가 영어로 바뀐다.
4. 로그아웃 후 다시 로그인한다.
5. 앱이 다크 모드와 영어 언어 설정 상태로 열린다.

**결과**
이지은의 테마 및 언어 설정이 저장되어, 다음 로그인 시에도 동일하게 적용된다.

#### 페르소나 B — 김민준: 테마 모드 복원

**배경**
김민준은 회사 PC에서 다크 모드를 사용하다가 고객사 미팅에서 화면을 공유할 일이 생겨 라이트 모드로 전환하려 한다.

**목표**
다크 모드에서 라이트 모드로 변경하고 저장한다.

**행동 흐름**
1. 환경설정 화면(S-08)으로 이동한다.
2. 테마 모드를 'Light Mode'로 변경한다. 화면이 즉시 밝은 테마로 바뀐다.
3. 변경된 설정이 자동으로 저장된다.
4. 미팅이 끝난 뒤 다시 환경설정 화면(S-08)으로 이동해 'Dark Mode'로 재설정한다.

**결과**
김민준은 상황에 따라 테마를 자유롭게 전환할 수 있고, 변경 사항은 즉시 저장된다.
